What Is HTML?
Let's start with what HTML isn't.
HTML is not a programming language. It is a Markup Language. In fact, HTML means, HyperText Markup Language. Markup languages are basically used to define a structure for content that tend to loose structure overtime. Keeping that in mind, a markup language is void of logic and other programming fundamentals. It is very close to WYSIWYG system but it isn't but it's close. That said, Let's talk about what HTML is.

HTML is one of three important technologies to learn if you ever hope to become a web developer. The other two are CSS and JavaScript. Now some of you who like to go faster than the teacher might might want to quickly go and see what CSS is like but they are put in that order for a reason. You cannot do CSS without knowing HTML and you definitely will not be able to rock javascript without enough css and html skills. That said, lets quickly look at how these three technologies work hand in hand to create beautiful web pages.

HTML defines the schema of a webpage. It defines where each visual element of your will sit in a top-down order. Visual elements your html can introduce includes and are not limited to Text, Images, Videos e.t.c. Remember, a markup language does only one thing well and that is defining a structure. At the end of the day, your pure html is void of look and feel. Either custom or default. Which is where CSS comes in. CSS means Cascaded StyleSheet. CSS introduces look and feel to your ugly looking html. From introducing colors to changing fonts, better defined layout, even animations and a whole bunch of amazing stuff which unfortunately is still not enough. Yes, our website now has a good look but it's missing one thing. User interaction which is core to creating better user experiences. And right there is where JavaScript comes in. JavaScript was bestowed the power to manipulate both HTML and CSS on the go. Meaning unlike HTML and CSS which only render what you the developer asks it to, JavaScript allows you to do to your already defined HTML and CSS what your user wants.
